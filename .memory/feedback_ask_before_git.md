---
name: ask before git commit or push
description: Always ask before committing or pushing — never do it automatically
type: feedback
---

Always ask for explicit confirmation before running `git commit`, `git push`, or any
destructive git operation. Do not stage, commit, or push even when a task appears complete.

**Why:** User wants full control over what goes into git history and when.
**How to apply:** When done with changes, summarize what was done and ask if they want
to commit, rather than committing automatically.
